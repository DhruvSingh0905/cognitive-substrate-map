Optimizing the Brain for Learning — a Network-Control Approach

Turning a biomedical knowledge graph into a signed, directed control system, and asking a control-theory question of it: which genes do you nudge — and by how much — to move the brain toward a learning-optimal state, at minimal systemic cost?

01The premise

Biology isn't a spreadsheet. Genes regulate other genes through transcription factors and signalling cascades, so cognition emerges from a network, not from any single gene. If you model that network as a graph, "how do I improve learning?" becomes a precise, two-part engineering question:

(1) Map the regulatory network of cognition as a directed, signed graph. (2) Find the optimal set of node perturbations — gene up/down-regulations — that cascade the rest of the network into a desired ("learning-optimal") state while minimizing off-target movement. It's a control problem.

02The substrate: a knowledge graph, scoped to the brain

The base is PrimeKG — 129,375 nodes and ~4M edges across ten types (genes/proteins, drugs, diseases, pathways, anatomy, phenotypes, biological processes, …). The lineage is Hetionet / Rephetio, a hetnet built to repurpose drugs by scoring the metapaths that connect a compound to a disease.

I re-purposed the idea for the brain: scope the whole-body graph down to the subnetwork that controls learning — the machinery around glutamate, BDNF, GABA, the cholinergic and dopaminergic systems — using tissue-expression enrichment, then bound it. A whole-body simulation is chaos; a focused core is tractable.

The result is 280 nodes / 3,912 directed edges (2,871 signed: 2,253 activating, 618 inhibitory), in four layers — intervention (the knobs), input (systemic drivers), readout (disease gauges), and a bounded one-hop expansion — with a single 200-node strongly-connected feedback core.

03From graph to operator

Each edge carries a direction and a sign — $+1$ for activation (a gas pedal), $-1$ for inhibition (a brake). Stack them into a weight matrix oriented so that multiplying by a state vector flows influence source → target: $\;W_{ij}=\operatorname{sign}(j\to i)$.

The hub problem, and the PageRank fix

A promiscuous regulator that touches 40 targets would otherwise dominate everything. The fix is a column-normalization by out-strength — the same transition operator behind Google's PageRank:

$$\hat{W}_{ij} = \frac{W_{ij}}{\sum_k \lvert W_{kj}\rvert}$$

Now a hub's outgoing influence is split across its targets: a gene shouting at 40 people is quieter, per target, than one whispering to a single partner. Crucially, the absolute-value matrix is column-stochastic, so the operator's spectral radius obeys $\rho(\hat W)\le 1$ — which guarantees the propagation in §4 converges. (The symmetric GCN normalization $\hat W = D^{-1/2}\tilde W D^{-1/2}$ is the undirected analogue; it damps the receiver rather than splitting the sender. For a directed regulatory graph the PageRank form is the right one.)

04Propagation: random-walk-with-restart

To see what a perturbation does, inject a signal $p$ (push one gene) and let it diffuse — but with a restart probability $\alpha$ that re-anchors the signal at its source each step. This is a random walk with restart, identical to Personalized PageRank / APPNP:

$$x^{(t+1)} = (1-\alpha)\,\hat{W}\,x^{(t)} + \alpha\,p$$

At the fixed point this has a closed form — the resolvent, equivalently a damped power series over hop-count $k$:

$$x^{*} = \alpha\bigl(I-(1-\alpha)\hat{W}\bigr)^{-1}p \;=\; \alpha\sum_{k\ge 0}(1-\alpha)^{k}\hat{W}^{k}p$$

Two things fall out of this. First, the $(1-\alpha)^k$ term is an over-smoothing guard: deep hops are exponentially suppressed, so $x^*$ never collapses to a uniform blob (the failure mode of naïvely deep graph diffusion). Second, because the substrate has a 200-node feedback loop, there is no "one-shot" answer — you must solve the steady state under feedback, which is exactly what the resolvent computes.

05Can you even steer it? Controllability

Before optimizing, a prior question: is this system controllable — can a handful of inputs drive it to an arbitrary state? Classical (Kalman) controllability asks whether the controllability matrix, for input matrix $B$, spans the whole state space:

$$\mathcal{C} = [\,B,\ WB,\ W^{2}B,\ \dots,\ W^{N-1}B\,]$$

For a large network, testing rank directly is impractical, so I used structural controllability (Liu–Barabási): the minimum number of driver nodes equals the number of nodes left unmatched by a maximum matching on the bipartite representation of the graph.

$$N_D = \max\!\left(N - |M^{*}|,\ 1\right)$$

where $M^{*}$ is a maximum matching. Intuitively, the denser the wiring, the larger the matching, and the fewer independent drivers you need — $\langle k\rangle \propto 1/N_D$. Running this on the substrate: all 280 nodes are reachable from the druggable set, full arbitrary-state control would need 81 independent drivers (only ~35 of them druggable) — but target-control of the cognitive core is plausible, which is all the optimizer actually needs. The 200-node feedback core is the structural signature of homeostasis: you can't just "set BDNF high" and walk away — the loop pushes back.

06The target: what "optimal" actually means

The least obvious part. I hand-curated a 94-gene target vector $d$ from a literature sweep, and the dominant finding is that most cognitive variables are not "maximize" — they are set-points with an inverted-U (Yerkes–Dodson / Arnsten): dopamine, arousal, cortisol, E/I balance, mTOR. 43 of 94 targets are inverted-U. A naïve "turn up the good genes" objective gets nearly half the network's direction wrong. A second finding: some genes are readouts (activity gauges like the immediate-early genes), not levers — tuning them is as useless as tuning a hub.

07Scoring an intervention

With a target vector $d$ and a computed response $x^{*}$, benefit is distance-reduction toward the target — which naturally rewards partial progress and penalizes overshoot (the far arm of the inverted-U):

$$b_i = \lvert d_i\rvert - \lvert x^{*}_i - d_i\rvert$$

Aggregate benefit weights each node by our confidence in its target; cost prices intervention effort (an $L_1$ term that rewards using few knobs) plus systemic collateral (movement in off-target nodes — the side-effect proxy):

$$B(p)=\sum_i w_i\,b_i, \qquad C(p)=\lVert p\rVert_1 + \gamma\!\!\sum_{j\,\notin\,\text{targets}}\!\! \lvert x^{*}_j\rvert$$

08Uncertainty and optimization

Because many edge magnitudes are sign-only and some targets are low-confidence, a single number would be dishonest. I propagate the input uncertainty by Monte-Carlo (the GUM-S1 procedure): sample the uncertain inputs, re-run, and report a coverage band on each intervention's score rather than a point. A Sobol sensitivity analysis then attributes the band — telling you which unknown to go verify to shrink it.

The optimization itself is genuinely multi-objective (maximize $B$, minimize $C$), so the honest output is a Pareto front, not a single winner. Because the system is small, I trace it exactly with the ε-constraint method ($\max B$ s.t. $C\le\varepsilon$, sweeping $\varepsilon$) rather than a weighted sum — which, on a non-convex problem, would silently miss parts of the front.

10The honesty ladder

Every layer sits on a different rung of groundedness, and saying so is the point: brain-scoping is computed; edge direction + sign is curated from causal databases; edge magnitude is mostly sign-only (a handful are real numbers); the objective and optimizer are specified and math-checked but consume a hand-built target vector. And the whole propagation is a linear response to a nonlinear system — valid for modest perturbations near baseline. So the deliverable is a ranking of interventions and a map of the trade-offs, not fold-change predictions, and certainly not a protocol. It's a math game — a serious one, but a game.
